# V187 운영 적용 순서

## 1. Supabase 적용

Supabase SQL Editor에서 다음 파일을 1회 실행합니다.

```text
supabase/migrations/20260710_v187_coupon_automation.sql
```

생성되는 표:

- `coupon_automation_retries`: 30분 뒤 최종 재시도
- `coupon_automation_failures`: 미확인 실패, 수동 재실행, 확인 완료

## 2. Cloudflare·GitHub 배포

1. V187 전체 소스를 GitHub `main` 브랜치에 반영합니다.
2. GitHub Actions의 검증과 Worker 배포 성공을 확인합니다.
3. Cloudflare Pages가 새 웹 빌드를 반영했는지 확인합니다.
4. 브라우저에서 `Ctrl + Shift + R`로 강력 새로고침합니다.

Worker Secret은 기존 `SUPABASE_URL`, `SUPABASE_SERVICE_ROLE_KEY`를 유지합니다.

## 3. Ncloud 최소 게이트웨이 갱신

V187 쿠폰 자동운영 API가 포함된 별도 `NCLOUD_FIXED_IP_GATEWAY_V187_20260710.zip`만 Ncloud에 적용합니다. 웹·R2 파일은 Ncloud에 저장하지 않습니다.

기존 `.dev.vars`를 복사한 뒤 다음 값이 있는지 확인합니다.

```text
COUPANG_COUPON_PREFLIGHT_ITEM_LIMIT=100
ALLOW_LIVE_EXTERNAL_API=true
ALLOW_FINAL_EXECUTION=true
ALLOW_SCHEDULED_WRITES=true
```

## 4. 최초 쿠폰 등록

1. 쿠폰관리에서 쿠팡 계약·쿠폰 목록을 조회합니다.
2. 반복할 쿠폰 여러 개를 직접 체크합니다.
3. `선택 쿠폰 일괄 반영`을 누릅니다.
4. `사전검증`을 실행합니다.
5. 통과·실패 사유와 다음 쿠폰명, 시작·종료시간을 확인합니다.
6. `자동운영 활성화`를 눌러 최종 확정합니다.
7. 활성화 즉시 기존 쿠폰을 변경하지 않고 당일 23:50부터 자동교체합니다.

## 5. 매일 자동 흐름

```text
23:45 사전점검
23:47 실패 쿠폰 재점검
23:49 최종 재점검
23:50 사전점검 통과 쿠폰별 기존 쿠폰 취소
23:51 취소 성공 쿠폰별 신규 쿠폰 생성·상품 적용
다음 날 23:50 신규 쿠폰 자체 종료 및 취소 확인
```

취소가 지연되면 23:55까지 보완하며, 취소 성공 직후 해당 쿠폰의 신규 생성·적용을 시작합니다. 생성·적용은 23:56까지 보완합니다.

## 6. 실패 처리

- 취소: 1차 즉시, 2차 10초 후, 3차 30초 후
- 생성·적용: 1차 즉시, 2차 10초 후, 3차 30분 후
- 생성됐지만 적용되지 않은 쿠폰은 먼저 취소·정리
- 최종 실패 쿠폰만 중단
- 앱 상단과 쿠폰관리 화면에 미확인 실패 표시
- `수동 재실행`은 현재 쿠팡 상태 확인 후 실패 단계부터 재개
- `확인 완료`를 눌러도 실패 운영이력은 보존

## 7. 자동운영 중지

`자동운영 중지`는 향후 작업과 Supabase 대기 재시도를 취소합니다. 현재 활성 쿠폰은 즉시 취소하지 않고 자체 종료시각까지 유지합니다.

## 8. 토스

토스 주문수집·상품준비중 조회·송장등록은 기존과 동일합니다. 공개 API에서 쿠폰 생성·취소 기능이 확인되기 전에는 토스 쿠폰 자동운영 버튼을 활성화하지 않습니다.
