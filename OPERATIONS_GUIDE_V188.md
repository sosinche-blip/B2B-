# V188 운영 적용 순서

## 1. 배포

1. V188 전체 소스를 GitHub `main` 브랜치에 반영합니다.
2. GitHub Actions의 `Deploy Cloudflare Worker`가 성공하는지 확인합니다.
3. Cloudflare Pages 최신 배포가 성공하는지 확인합니다.
4. 브라우저에서 `Ctrl + Shift + R`로 강력 새로고침합니다.

기존 Ncloud V187 고정 IP 게이트웨이는 변경하지 않습니다.

## 2. 간편운영

1. 앱 실행 후 쿠팡·토스 현황이 자동 조회됩니다.
2. 필요하면 `현황 새로고침`을 누릅니다.
3. `쿠팡 주문조회` 또는 `토스 주문조회`를 누릅니다.
4. 결제완료 상품 목록에서 필요한 항목만 체크합니다.
5. `선택 주문 수집`을 눌러 매핑·발주파일 생성을 진행합니다.

## 3. 업체 바로가기

1. B2B 바로가기 설정에는 업체명, URL, 로그인ID를 저장합니다.
2. 모바일에서 업체 버튼을 누르면 업체 로그인 페이지가 열립니다.
3. 로그인ID는 클립보드에 복사됩니다.
4. 비밀번호는 모바일 브라우저 비밀번호 관리자의 자동완성을 사용합니다.

비밀번호는 앱·Supabase·GitHub에 저장하지 않습니다.

## 4. 쿠폰관리

쿠폰 선택·사전검증·자동운영 활성화·중지·삭제는 Supabase에 자동 저장됩니다. 별도의 브라우저 저장 또는 서버 저장 버튼을 사용하지 않습니다.
