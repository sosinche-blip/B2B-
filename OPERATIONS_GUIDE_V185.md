# V185 운영 적용 순서

## 1. 배포

1. V185 소스를 GitHub `main` 브랜치에 반영합니다.
2. GitHub Actions 또는 로컬에서 웹 빌드와 Worker 타입검사를 완료합니다.
3. Cloudflare Pages와 Worker를 배포합니다.
4. Ncloud에는 V185 전체 ZIP을 올리지 않습니다. 현재 실행 중인 고정 IP API 게이트웨이만 유지합니다.
5. 기존 R2 버킷은 발주·결과파일 등 다른 기능에 사용할 수 있지만, 업체 송장 임시선택 흐름에서는 사용하지 않습니다.

## 2. 송장 처리

1. 업체별 송장 엑셀에 택배사와 운송장번호를 입력합니다.
2. `업체송장 선택`을 눌러 여러 파일을 선택합니다.
3. 파일은 브라우저 앱 메모리에만 임시 보관됩니다.
4. `쿠팡+토스 업로드`를 누릅니다.
5. 앱이 쿠팡·토스 상품준비중 주문을 조회하고 다음 순서로 매칭합니다.
   - 주문번호 강제 매칭
   - 성명+주소 앞 2단어
   - 성명
   - 중복 후보는 상품명 2글자 이상 일치
6. 화면에서 쿠팡 준비, 토스 준비, 확인필요 건수를 검토합니다.
7. `최종 업로드`를 누르면 정상 매칭 건만 채널별로 등록합니다.
8. 미매칭 건은 업로드하지 않고 `미매칭_확인.xlsx`에 포함합니다.
9. 결과 엑셀 4개와 전체 ZIP이 다운로드됩니다.

## 3. 다운로드 파일

- `쿠팡_송장업로드결과.xlsx`
- `토스_송장업로드결과.xlsx`
- `미매칭_확인.xlsx`
- `전체_처리결과.xlsx`
- `B2B_송장업로드결과_날짜시간.zip`

## 4. 안전 기준

- 파일 선택만으로 외부 API를 실행하지 않습니다.
- 첫 번째 `쿠팡+토스 업로드`는 매칭 미리보기만 수행합니다.
- 실제 API 반영은 `최종 업로드`에서만 수행합니다.
- 일부만 매칭된 경우 정상 매칭 건만 등록하고 나머지는 별도 엑셀로 제공합니다.
- API 오류 시 임시 송장과 미리보기를 유지해 원인 확인 후 재시도할 수 있습니다.
- 정상 완료 후 임시 송장은 앱에서 삭제됩니다.
- 새로고침·브라우저 종료 시 임시 송장은 삭제됩니다.
