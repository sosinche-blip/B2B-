# V184 운영 적용 순서

## 1. 배포

1. Cloudflare R2 버킷 `b2b-operation-files`와 Worker 바인딩 `B2B_FILES`를 유지합니다.
2. V184 소스를 GitHub `main` 브랜치에 반영합니다.
3. GitHub Actions에서 웹 빌드, Worker 타입검사, Worker 배포가 성공했는지 확인합니다.
4. Ncloud는 현재 실행 중인 고정 IP API 게이트웨이만 유지합니다. V184 전체 ZIP을 Ncloud에 올리지 않습니다.

## 2. 송장 처리

1. 업체에서 내려받은 송장 엑셀의 택배사와 운송장번호를 확인합니다.
2. 앱의 `업체송장 선택` 버튼을 눌러 여러 파일을 선택합니다.
3. 앱이 파일을 Cloudflare R2 발주폴더로 저장합니다.
4. `쿠팡+토스 업로드` 버튼을 누릅니다.
5. 앱이 최근 상품준비중 주문과 업체 송장을 매칭합니다.
6. 매칭 우선순위는 주문번호 → 성명+주소 앞 2단어 → 성명 → 상품명 2글자 이상입니다.
7. 등록준비 건을 쿠팡과 토스에 각각 업로드합니다.
8. 관련 업체 송장, 채널 운송장 입력파일, 송장등록 확인표를 ZIP으로 자동 다운로드합니다.

## 3. 안전 기준

- 등록준비가 0건이면 외부 업로드와 자동 ZIP 다운로드를 실행하지 않습니다.
- 확인필요 건은 송장등록 확인표에서 수정합니다.
- ZIP은 이번 처리와 관련된 파일명만 지정해 생성하며 과거 발주폴더 전체를 무조건 포함하지 않습니다.
