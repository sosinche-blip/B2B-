# V189 운영 가이드

## 1. 기존 반복쿠폰 금액 변경

1. 쿠폰관리의 `24시간 반복대상 목록`으로 이동합니다.
2. 할인구분과 할인값을 수정합니다.
3. 정률 쿠폰은 최대 할인금액을 입력합니다.
4. 필요하면 와우회원 전용 여부를 변경합니다.
5. `다음 발행부터 적용`을 누릅니다.
6. 상단 `사전검증`을 다시 실행합니다.

현재 활성 쿠폰은 그대로 유지되며 다음 23:51 신규 생성분부터 변경값이 적용됩니다.

## 2. API 옵션ID 조회

1. `계약서 목록 조회` 후 사용할 계약서의 `신규생성 선택`을 누릅니다.
2. `API 옵션ID 조회·신규 쿠폰 등록` 영역에 쿠팡 vendorItemId를 입력합니다.
3. 여러 옵션은 줄바꿈 또는 쉼표로 구분합니다.
4. `API 옵션ID 조회`를 누릅니다.
5. `API 확인`으로 표시된 옵션만 체크합니다.

판매가·상태·재고는 쿠팡 API 값이며, 상품명·옵션명은 앱에 저장된 주문·매핑·옵션자료에서 보완됩니다.

## 3. 목록에 없는 신규 쿠폰 등록

1. contractId를 확인합니다.
2. 신규 쿠폰명과 할인조건을 입력합니다.
3. 정률 쿠폰은 최대 할인금액을 입력합니다.
4. 시작·종료일시를 비워두면 23:51~다음 날 23:50 권장시간을 사용합니다.
5. `신규 쿠폰 사전검증`을 누릅니다.
6. 통과 후 `신규 쿠폰 생성·적용`을 누릅니다.
7. 생성된 couponId가 반복 초안으로 자동 저장됩니다.
8. 기존 상단 `사전검증`을 실행한 뒤 `자동운영 활성화`를 누릅니다.

## 4. 안전 원칙

- 생성 버튼은 사전검증을 통과해야 활성화됩니다.
- API에서 확인되지 않은 옵션ID는 생성 대상에서 제외됩니다.
- 정액 할인액이 판매가 이상이면 차단됩니다.
- 같은 날짜·같은 이름의 활성 또는 대기 쿠폰이 있으면 차단됩니다.
- 현재 활성 쿠폰의 금액은 직접 수정하지 않습니다.
- 쿠폰 API 실행은 Ncloud 고정 IP 게이트웨이를 통해 진행합니다.
