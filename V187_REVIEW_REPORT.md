# V187 전반 검토 보고

## 1차 기능 검토

- 사용자가 체크한 여러 쿠팡 쿠폰만 자동운영 대상으로 저장
- 실제 변경 없는 사전검증과 최종 활성화 분리
- 23:45·23:47·23:49 검증, 23:50 취소, 23:51 생성·적용 흐름 확인
- 계약ID, 상품·옵션, 할인조건, 최대할인금액, 와우 여부, 쿠폰명 날짜 복사 확인
- 쿠폰별 독립 처리와 실패 쿠폰만 중단 확인
- 자동운영 중지 시 대기 재시도 취소 확인

## 2차 안전성 검토

- 쿠팡 쿠폰 생성·파기·아이템 적용의 비동기 `requestedId` 상태를 DONE/FAIL까지 확인
- 처리 중 상태에서는 신규 쿠폰을 다시 생성하지 않고 30분 뒤 상태 확인만 예약
- 일부 옵션 실패 수량이 있으면 전체 쿠폰 실패로 처리
- 생성 후 적용 실패 쿠폰 정리와 정리 실패 재시도 확인
- 같은 날짜·쿠폰·단계 중복 실행 방지 확인
- 허용시간 초과 시 미확인 실패 알림 생성 확인

## 3차 배포·회귀 검토

- 웹 프로덕션 빌드 통과
- Worker TypeScript 검사 통과
- Ncloud 최소 게이트웨이 빌드 및 로컬 상태 API 실행 통과
- 기존 주문수집·매핑·발주·송장 임시매칭·통합 결과파일 기능 유지
- Ncloud 로컬 파일·UI 저장 코드 제거 확인
- 실제 인증정보·로그·node_modules·dist 제외 후 배포 압축

## 토스 검토

토스쇼핑의 공개 API 문서 범위에서 상품·주문·재고·배송 연동은 확인되지만 쿠폰·프로모션 생성·취소 API는 확인되지 않았습니다. 따라서 V187은 토스 쿠폰 자동운영을 명시적으로 비활성화하며 임의 엔드포인트나 브라우저 자동조작을 사용하지 않습니다.
