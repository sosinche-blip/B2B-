# V170 모바일 매핑 운영 안정화

## 핵심 목적

V170은 쿠팡·토스 API 연결 성공 이후 실제 모바일 운영에 필요한 매핑·발주 다운로드 흐름을 정리한 버전입니다. PC 로컬폴더 저장은 보조 기능으로 두고, 모바일과 Cloudflare Pages 환경에서는 브라우저 ZIP 다운로드와 Supabase 설정 저장을 기본 흐름으로 사용합니다.

## 주요 변경

1. 모바일 운영 문구와 메뉴 흐름 정리
   - 주문수집 → 매핑확인 → 발주 ZIP 다운로드 → 송장처리 흐름을 기본 안내로 수정했습니다.
   - PC 전용 START_HERE_WINDOWS.cmd, 폴더 열기, 로컬 저장 기능은 보조 기능임을 명확히 했습니다.

2. 매핑 엑셀 처리 보강
   - V170 매핑 양식 다운로드를 추가했습니다.
   - `발주처`, `공급처`, `거래처`, `발주상품명`, `발주수량배수`, `사용여부` 같은 실무 컬럼명을 인식하도록 보강했습니다.
   - 사용여부가 N/미사용/중지/false/0이면 업로드 시 제외됩니다.

3. 발주파일 다운로드 개선
   - 전체 발주 생성 시 업체별 발주파일과 검증표를 하나의 ZIP으로 내려받을 수 있도록 보강했습니다.
   - PC 로컬폴더 helper가 없거나 모바일 환경이면 자동으로 브라우저 ZIP 다운로드로 전환합니다.

4. 검증
   - `npm run verify:all` 통과
   - Web production build 통과
   - Worker TypeScript check 통과
   - V170 service verification 통과

## 운영 전제

- 쿠팡·토스 API 호출 IP는 네이버클라우드 공인 IP `101.79.27.234`를 기준으로 운영합니다.
- 현재 Quick Tunnel 주소는 임시 테스트용입니다. 실운영 전 고정 Cloudflare Tunnel 또는 도메인 연결이 필요합니다.
- 매핑 엑셀 실제 자료가 올라오면 컬럼 구조를 확인해 최종 표준 매핑 양식에 맞춰 확정해야 합니다.
