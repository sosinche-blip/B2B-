# V175 서버 매핑저장 안정화

## 목적

V174에서 Ncloud CORS와 직접터널 연결은 정상화되었지만, 매핑관리 화면의 `서버 저장`에서 `/api/operation/settings/save`가 HTTP 500으로 실패하는 상황을 보완했습니다.

## 핵심 수정

- 서버 영구저장은 운영에 꼭 필요한 매핑/양식 설정 중심의 compact payload를 우선 전송합니다.
- 발주 이력, 쿠폰 이력, PC 로컬 경로 등 계속 커지는 자료는 서버 저장 대상에서 제외하고 브라우저 저장에 유지합니다.
- 서버 저장 API는 1차 저장 실패 시 매핑 중심 compact payload로 자동 재시도합니다.
- 저장 성공 응답에 매핑 건수, 토스 옵션ID 건수, 쿠팡 옵션마스터 건수 등 summary를 반환합니다.
- 저장 실패 시 Supabase 오류 메시지를 JSON으로 반환하여 원인 확인이 가능하도록 했습니다.

## 정상 목표

- 매핑관리 → 매핑 엑셀 업로드
- 브라우저 저장 성공
- 서버 저장 성공
- 최신 불러오기 시 전체 매핑 건수 유지
- 주문 수집 후 저장된 일부 매핑만 적용되는 문제 방지

## 서버 적용 필요

V175는 `apps/web`과 `apps/worker`, `scripts/ncloud_node_server.ts`가 모두 변경되었습니다.
따라서 GitHub/Pages 재배포와 Ncloud 서버 파일 교체 후 재시작을 모두 진행해야 합니다.
