# V172_MOBILE_OPERATION_GUARD 변경 내역

## 1. 개발 목적

V170에서 모바일 중심 운영 흐름을 반영한 뒤, V172에서는 실제 일일 운영 중 순서가 섞이지 않도록 `모바일 단계 잠금판`을 추가했습니다. 쿠팡·토스 주문 수집 이후 매핑, 발주 ZIP, 업체 발주 업로드 확인, 업체 송장 업로드, 송장 매칭, API 업로드, 쿠폰 자동화, 서버 정리까지 9단계를 상태값으로 점검합니다.

## 2. 주요 변경사항

1. 앱 버전명을 `V172_MOBILE_OPERATION_GUARD`로 변경했습니다.
2. 간편운영 화면에 `V172 모바일 단계 잠금판`을 추가했습니다.
3. 단계별 상태를 `완료`, `진행`, `대기`, `확인필요`로 표시하도록 했습니다.
4. 사용자가 앱 밖에서 수행하는 4단계 `업체 사이트 발주 업로드 완료`, 5단계 `업체 송장 엑셀 수령 완료`를 모바일 체크박스로 기록할 수 있게 했습니다.
5. 단계점검표를 엑셀 파일로 내려받을 수 있는 `단계점검표` 버튼을 추가했습니다.
6. 발주 ZIP, 업체송장 업로드, 쿠팡+토스 업로드 버튼을 단계 잠금판에 배치해 모바일에서 다음 실행 버튼을 찾기 쉽게 정리했습니다.
7. 쿠팡·토스 송장 업로드 단계가 아직 대기 상태일 때 바로 실행하지 않도록 버튼 비활성 조건을 추가했습니다.
8. 간편운영 지표의 주문 수량 중복 표시를 수정했습니다.
9. 매핑 양식·매핑 자료 파일명을 V172 기준으로 변경했습니다.

## 3. 유지된 핵심 운영 순서

1. 쿠팡/토스 주문 수집
2. 옵션ID 기준 자동 매핑
3. B2B 업체별 발주 엑셀 생성
4. 사용자가 업체에 발주 엑셀 업로드
5. 업체별 송장 엑셀 업로드
6. 주문번호 → 성명+주소 앞 2단어 → 성명 → 상품명 2글자 이상 일치 순으로 송장 매칭
7. 쿠팡·토스 송장 업로드 파일 생성 및 API 업로드
8. 쿠팡 즉시할인쿠폰 23:50 일괄 취소, 23:51 일괄 적용
9. 스케줄러와 서버 저장용량 자동 정리

## 4. 검토 결과

1. 1차 검토: V170 요구사항 유지 여부 확인
   - 주문수집, 옵션ID 매핑, 업체별 발주 ZIP, 송장 매칭, 쿠폰 자동화, 서버 정리 흐름을 유지했습니다.

1.1 2차 검토: 모바일 운영성 확인
   - 사용자가 앱 밖에서 직접 처리해야 하는 업체 발주 업로드와 송장 수령 단계를 체크박스로 보조했습니다.
   - 핵심 실행 버튼을 단계 잠금판에 모아 모바일 스크롤 이동을 줄였습니다.

① 3차 검토: 배포 전 안정성 확인
   - Web production build와 Worker TypeScript check 대상 파일을 V172 기준으로 정합화했습니다.
   - 이전 V170 문서는 참고용으로 유지하되, V172 운영 문서를 신규 작성했습니다.
