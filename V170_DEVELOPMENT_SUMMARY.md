# V170_MOBILE_MAPPING_OPERATION 개발 요약

## 1. 반영 목표

1. 쿠팡·토스 API 주문 수집 성공 상태를 전제로, 이후 운영 흐름을 모바일 중심으로 재설계했습니다.
1.1 PC 로컬폴더 중심 기능은 보조 기능으로 낮췄습니다.
1.2 매핑 엑셀 업로드, 미매핑 카드형 수정, 업체별 발주 엑셀 ZIP 다운로드, 업체 송장 엑셀 업로드, 쿠팡·토스 송장 업로드, 쿠폰 자동화, 서버 저장용량 정리 흐름을 V170 기준으로 정리했습니다.

## 2. 핵심 변경

1. 화면 버전명
- `APP_VERSION = "V170_MOBILE_MAPPING_OPERATION"`으로 변경했습니다.

1.1 모바일 운영 순서
- 간편운영 화면에 9단계 모바일 운영 순서 카드를 추가했습니다.
- 순서: 주문 수집 → 옵션ID 매핑 → 업체별 발주 ZIP → B2B 업체 발주 → 업체 송장 업로드 → 송장 자동 매칭 → 쿠팡·토스 송장 등록 → 쿠폰 자동화 → 서버 정리.

1.2 미매핑 카드형 수정
- 미매핑 주문을 카드로 보여주고, 업체명·업체상품명·코드·원가·기본수량을 바로 입력해 매핑으로 저장할 수 있게 했습니다.
- 저장 기준은 채널+옵션ID입니다.

1.3 발주 엑셀 ZIP 다운로드
- `전체 발주` 기능을 모바일 기본 흐름에 맞춰 업체별 발주 엑셀과 검증표를 하나의 ZIP으로 다운로드하도록 변경했습니다.
- PC 로컬폴더 저장·열기는 보조 기능으로 유지했습니다.

1.4 송장 매칭 기준
- 송장 매칭 보조 기준을 명시·보강했습니다.
- 적용 순서: 주문번호 → 성명+주소 앞 2단어 → 성명 → 상품명 2글자 이상.

1.5 쿠폰·스케줄러·서버 정리
- 쿠팡 쿠폰 23:50 취소, 23:51 적용 흐름을 유지했습니다.
- 서버 저장용량 점검·만료자료 정리 스케줄러를 유지했습니다.

## 4. 3회 검토 결과

1. 1차 검토: 사용자 작업순서 반영 여부
- 요청한 1~9단계 흐름이 화면·기능·문서에 반영됐는지 확인했습니다.

1.1 2차 검토: 모바일 운영 적합성
- PC 로컬폴더 저장 중심 문구를 줄이고, 업로드·ZIP 다운로드 중심으로 정리했습니다.
- PC 로컬폴더는 보조 기능으로 남겼습니다.

1.2 3차 검토: 빌드 및 실행 안정성
- Web build와 Worker typecheck를 통과했습니다.
- package/package-lock 버전을 V170으로 정합화했습니다.
