# V194 5회 점검 결과

1. 상태 선택 점검
   - 결제완료 프리셋은 쿠팡 `ACCEPT`, 토스 `PAID`를 사용합니다.
   - 상품준비중 프리셋은 쿠팡 `INSTRUCT`, 토스 `PREPARING_PRODUCT`를 사용합니다.
   - 주문조회가 더 이상 결제완료 상태로 강제 고정되지 않고 현재 선택 상태를 사용합니다.

2. 조회·선택 점검
   - 결제완료와 상품준비중 모두 조회목록에 표시됩니다.
   - 선택수집 패널 제목·안내·목록 상태가 선택한 상태에 맞게 표시됩니다.
   - 상태 없는 호환 응답은 요청한 모드를 기준으로 누락 없이 표시합니다.

3. 수합 구조 점검
   - 기존 주문 전체를 채널 단위로 삭제하지 않습니다.
   - 주문번호·옵션ID가 같으면 상태·주소를 최신값으로 갱신합니다.
   - 신규 주문은 추가하고 동일 데이터는 중복 제외합니다.

4. 업무 분기 점검
   - 결제완료는 기존 업체별 발주 ZIP과 상품준비중 변경 흐름을 유지합니다.
   - 상품준비중은 송장매칭·업로드 대상에만 반영합니다.
   - 상품준비중 수합에서 발주파일 생성과 상태변경 API를 호출하지 않습니다.

5. 실행·회귀 점검
   - 상품준비중 선택수합 정적 회귀검증 통과
   - 주소 무결성 회귀검증 통과
   - 웹 production build 통과
   - Worker TypeScript 검사 통과
   - V194 전체 검증 통과
