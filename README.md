# B2B Operation V187

쿠팡·토스 주문수집, 업체별 발주, 브라우저 임시 송장 매칭, 채널별 송장등록 기능에 **쿠팡 즉시할인쿠폰 24시간 자동운영 안전장치**를 추가한 운영본입니다.

## 1. 쿠팡 쿠폰 자동운영

- 쿠팡에 등록된 쿠폰 목록에서 사용자가 체크한 여러 쿠폰만 대상
- 실제 변경 없는 사전검증 후 `자동운영 활성화`로 최종 확정
- 한국시간 매일 23:45 사전점검, 23:47·23:49 재점검
- 23:50 기존 쿠폰별 취소, 취소 성공 쿠폰만 23:51 신규 생성·적용
- 신규 쿠폰 시작 23:51, 종료 다음 날 23:50
- 기존 계약ID·상품/옵션·할인방식·할인값·최대할인금액·와우 전용 여부 복사
- 쿠폰명 끝 날짜를 `YYYY-MM-DD`로 교체하거나 없으면 추가
- 쿠폰별 독립 처리, 중복실행 방지, 요청상태 재조회
- 취소 실패: 즉시·10초·30초, 생성/적용 실패: 즉시·10초·30분 뒤 최종 재시도
- 30분 재시도와 실패 알림은 Supabase에 저장
- 실패 쿠폰만 중단하고 정상 쿠폰은 계속 운영
- 자동운영 중지 시 대기 재시도를 취소하고 현재 쿠폰은 종료시각까지 유지

## 2. 토스 쿠폰 자동운영

토스쇼핑 공개 API의 현재 문서 범위에는 쿠폰·프로모션 생성/취소 API가 확인되지 않아 동일 자동화를 실제 호출하지 않습니다. 주문수집과 송장등록은 기존대로 유지하며, 공식 쿠폰 API가 제공될 때만 별도 활성화하도록 비활성 플래그를 유지합니다.

## 3. 역할 분리

- Cloudflare Pages: 모바일 웹 UI
- Cloudflare Worker: HTTPS 게이트웨이, R2 연동, 매분 스케줄 호출
- Supabase: 설정, 재시도, 실패 알림, 운영이력
- Ncloud: 고정 IP가 필요한 쿠팡·토스 API 실행과 쿠폰 스케줄 실행
- GitHub Actions: 검증과 Worker 배포
