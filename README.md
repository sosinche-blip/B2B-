# B2B Operation V189

V189는 V188 주문·송장·쿠폰 자동운영을 유지하면서 쿠폰관리의 누락 기능을 실제 화면에 추가한 버전입니다.

1. 쿠팡 API 옵션ID 조회
   - `vendorItemId`를 줄바꿈·쉼표로 최대 200개 입력
   - 기존 Ncloud 고정 IP 게이트웨이의 판매가 조회 API로 옵션 존재·판매가·상태·재고 확인
   - 상품명·옵션명은 앱의 주문·매핑·기존 옵션자료에서 보완해 표시
   - API 확인에 성공한 옵션만 신규 쿠폰 대상으로 선택
2. 목록에 없는 신규 쿠폰 등록
   - 계약ID, 쿠폰명, 정액·정률, 할인값, 최대 할인금액, 와우 전용, 운영기간 입력
   - 옵션ID API 조회 → 신규 쿠폰 사전검증 → 쿠폰 생성·옵션 적용
   - 생성된 couponId를 24시간 반복운영 초안으로 Supabase에 자동 저장
   - 이후 기존 `사전검증`과 `자동운영 활성화` 절차로 운영 시작
3. 기존 반복쿠폰 금액 변경
   - 반복대상 목록에서 할인구분·할인값·최대 할인금액·와우 전용 수정
   - `다음 발행부터 적용`을 눌러 Supabase 자동 저장
   - 현재 활성 쿠폰은 변경하지 않고 다음 23:51 신규 발행분부터 적용
   - 변경 후 사전검증 상태를 초기화해 재검증하도록 안전 처리
4. 기존 구조 유지
   - Ncloud V187 최소 고정 IP 게이트웨이는 변경하지 않음
   - Cloudflare Worker·Pages와 Supabase만 V189로 배포
   - 토스 쿠폰 자동화는 공개 API 부재로 계속 비활성화
