# V176 주문관리 단순화·수집초기화 안정화

## 목적

주문관리 화면에서 운영에 불필요한 PC 발주 폴더/발주파일 다운로드 영역을 제거하고, 주문수집을 다시 실행할 때 이전 수집 결과가 남아 다음 결과에 섞이지 않도록 자동 초기화합니다.

## 변경사항

1. 주문관리 상단의 발주 폴더 설정 영역 제거
2. 주문관리의 발주파일 PC·모바일 다운로드 패널 제거
3. 주문관리의 발주파일 목록 버튼 제거
4. 쿠팡 단독 수집 시 기존 쿠팡 주문결과를 초기화하고 새 결과로 교체
5. 토스 단독 수집 시 기존 토스 주문결과를 초기화하고 새 결과로 교체
6. 쿠팡+토스 수집 시 기존 전체 주문결과를 초기화하고 새 결과로 교체
7. 주문수집 시작 시 이전 진단표, 수집요약, 발주파일 생성표, 발주파일 목록 표시를 초기화
8. V175 서버 매핑저장 안정화와 Supabase 저장 로직 유지

## 적용 메모

- 서버 저장 로직은 V175와 호환됩니다.
- Ncloud 서버는 이미 V175/Node.js 22로 정상 저장되면 필수 재설치 대상은 아닙니다.
- 화면 버전 표시와 주문관리 UI 반영을 위해 GitHub/Cloudflare Pages 재배포가 필요합니다.
