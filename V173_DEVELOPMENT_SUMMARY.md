# V173_MOBILE_ENV_BINDING_GUARD 개발 요약

## 1. 버전명

`V173_MOBILE_ENV_BINDING_GUARD`

## 2. 반영 목적

허용 IP 등록이 완료된 상태에서도 쿠팡 주문수집이 실패하는 경우를 분리 진단하도록 개선했습니다. 이번 오류의 핵심은 IP가 아니라 `쿠팡 API 키 미주입` 가능성이 높으므로, 모바일 화면과 서버 응답 모두에서 환경변수 주입 상태를 먼저 확인할 수 있게 했습니다.

## 3. 주요 개발 항목

1. Worker 환경변수 진단 API 추가
   - 경로: `GET /api/system/env-diagnostics`
   - 쿠팡·토스·Supabase 키 상태 마스킹 표시
   - Gate 상태 표시
   - 런타임과 환경파일 출처 표시

2. 쿠팡 인증 미주입 오류 개선
   - `COUPANG_CREDENTIALS_MISSING` 구조화
   - `CONFIG_MISSING` 오류 분류 추가
   - 주문수집 진단표에 누락 키 항목 표시

3. Ncloud 환경파일 로딩 보강
   - `.dev.vars` 자동 탐색 범위 확대
   - 직접 빌드된 `.ncloud/api-server.mjs` 실행 시에도 환경파일 로딩 가능

4. 모바일 UI 개선
   - 운영설정 고급 메뉴에 `환경변수 점검` 버튼 추가
   - IP 확인 문구를 `등록필요` 중심에서 `등록완료 후 환경변수 확인` 중심으로 변경

## 4. 3회 검토 결과

1. 1차 검토: 화면상 IP 등록 안내가 원인을 혼동시킬 수 있어 문구를 수정했습니다.
2. 2차 검토: 쿠팡 주문수집 실패가 `HTTP 0`으로만 보이지 않도록 서버 응답을 구조화했습니다.
3. 3차 검토: Ncloud 실행 위치와 배포 방식 차이로 `.dev.vars`가 누락되는 경우를 막기 위해 런타임 환경파일 탐색을 추가했습니다.
